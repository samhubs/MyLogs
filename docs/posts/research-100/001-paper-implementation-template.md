# Day 001: Paper Implementation Template

<p class="post-meta">Research · Reproducibility · Implementation</p>

This is the working template for the 100-day research implementation series. I can duplicate this file for each paper, rename it with the day number and paper slug, then fill in the sections as the implementation develops.

<div class="tag-row">
  <span>Research</span>
  <span>Paper Reading</span>
  <span>Reproduction</span>
  <span>Experiment Log</span>
</div>

## Paper

- **Title:** Paper title
- **Authors:** Author list
- **Link:** Paper URL
- **Code:** Official implementation, if available
- **Dataset:** Dataset or benchmark
- **Goal:** What I want to reproduce

## Why This Paper?

Write the personal and technical reason for choosing the paper. This section should make the reader understand why the paper matters and why it belongs in the series.

## Core Idea

Explain the paper in my own words. Keep this section readable enough for an engineer who has not read the paper yet, but precise enough to show the actual mechanism.

## Implementation Plan

- Baseline to build first
- Components to implement
- Training or evaluation setup
- Metrics to track
- Expected result from the paper

## Reproduction Log

Use this section as the diary while working.

| Step | Status | Notes |
| --- | --- | --- |
| Read paper | Planned | Capture assumptions and claims |
| Build baseline | Planned | Start with the smallest working version |
| Run experiment | Planned | Save config, seed, and environment |
| Compare result | Planned | Match against reported metric or qualitative result |

## Results

Add metrics, plots, screenshots, tables, or links to code. If the reproduction fails, document that clearly. A failed reproduction with good notes is still useful research work.

## What I Learned

Capture the practical lesson:

- Modeling lesson
- Engineering lesson
- Debugging lesson
- Future experiment

## References

- Paper link
- Code link
- Dataset link
